Research

Abstract


The Problem

Emojis are becoming more and more popular as a means to express feelings, but the ever-expanding emoji library has added to the headache and hassle of finding the right emoji to communicate the desired message.

Up to this point, no solution can systematically address this growing problem. Most commercial applications only use keyword searching to recommend Emojis to users. Without taking the whole context into consideration, those algorithms often overlook important underlying messages from texts.

Our Solution

In response to this user experience headache, Blissmoji developed a keyboard that produces an exhaustive list of emoji recommendations based on the user’s text input. Using a machine learning algorithm trained on Tweets, we provide accurate recommendations with sentiment analysis and context-based prediction technologies for texters to find the emojis they want to use faster.

Data Processing


We extracted 5000 sentences containing the top 20 frequently used emojis from Twitter. To avoid the imbalanced dataset, we collect around 500 sentences for each emoji to equalize the number of sentences for each emoji. For a better representative of each emoji, we cleaned the dataset by removing the stop words, punctuation, user names, and links from sentences.

We split the dataset into a training dataset, validation dataset, and test dataset. 80 percent of the dataset is training.

Method


We tried three different types of machine learning models to do emoji prediction, such as model only with LSTM, model that has Bi-LSTM, and model that has both CNN and BI-LSTM.

Word Embedding

We used GloVe with 300 dimensions to obtain vector representation for words. GloVe is an unsupervised learning algorithm that captures as much as possible the meaning of a word to map two related words. In our situation, for the performance of the neural net models, a single scalar is problematic in quantifying the relatedness of two words. Thus, we used 300 dimensions vector to represent each word so that closed meaning words can be mapped to space where their distances are related to semantic similarity.

Model

First model: Embedding layer to LSTM layer to linear layer to SoftMax.

As mentioned above, we loaded a pre-trained 300 dimensions GloVe model to represent each word as vectors and this is also the meaning of the embedding layer. Moreover, since the LSTM layer by itself doesn’t take advantage of text structure, we need to combine it with this embedding layer.

LSTM stands for long short-term memory which is a type of recurrent neural network. The advantage of the LSTM is that it can remember things over time. In another word, it implements previous outputs as the inputs while having a hidden layer, so it fits sentence analysis.

Lastly, we have a linear layer which is a linear transformation of the dataset, and a SoftMax to do the prediction of the 20 emojis.

Second Model: embedding layer to Bi-LSTM layer to Linear layer to SoftMax.

Bi-LSTM stands for bidirectional LSTM and it preserves information from both past and future while LSTM only maintains past information. So it is more suitable for sentence analysis.

Third Model: embedding layer to CNN layer to Bi- LSTM to SoftMax.

We add an additional CNN layer and remove the Linear layer to reach a better model performance. The model output size is expanded to top five predictions.
